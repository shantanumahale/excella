description: Test-driven development workflow (RED -> GREEN -> REFACTOR)
---

Use the **tdd-guide** agent to enforce test-first development.

## Instructions

1. Understand the feature or fix being implemented
2. Write a failing test that describes expected behavior (RED)
3. Run the test - verify it FAILS
4. Write minimal implementation to make the test pass (GREEN)
5. Run the test - verify it PASSES
6. Refactor while keeping tests green (IMPROVE)
7. Verify 80%+ coverage
8. Repeat for each behavior/edge case
